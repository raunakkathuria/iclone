# memory

What the clone learns and keeps between sessions — facts about you, your preferences, corrections
you have made, ongoing context that should not reset each time.

Keep each memory small and specific: one fact per file, with a one-line index in a `MEMORY.md` so
the clone can scan what it knows. The point is that your corrections compound instead of being
re-explained every session.

Do not store anything here you would not want in a private repo — and never anything you would not
want in a public one if you are sharing this.
